# Claude 개발 가이드

이 프로젝트는 .NET Framework 4.8 기반의 WPF 애플리케이션 `SJMagic`입니다. 
클로드(Claude)는 이 프로젝트에서 작업할 때 다음 규칙과 워크플로우를 반드시 준수해야 합니다.

## 1. 아키텍처 및 코딩 패턴

*   **MVVM 패턴**: 
    *   모든 UI 관련 작업은 철저하게 MVVM(Model-View-ViewModel) 패턴을 준수하여 작성해야 합니다.
    *   View의 Code-Behind(예: `.xaml.cs`)에 직접적인 비즈니스 로직을 작성하지 마십시오. 필요한 경우 ViewModel의 ICommand 또는 프로퍼티 바인딩을 사용하세요.
*   **스타일 관리**:
    *   인라인 스타일(Inline Style) 사용을 지양하고, 반드시 **Resource Dictionary**를 생성하여 스타일을 딕셔너리로 관리해야 합니다.
    *   이를 통해 프로젝트 전체의 스타일을 중앙 집중식으로 관리하고 일관성을 유지합니다.
*   **객체지향 설계 (OOP) 및 Base 클래스**:
    *   새로운 클래스를 만들 때는 항상 객체지향 원칙을 고려합니다.
    *   상황에 따라 공통 기능을 추상화할 수 있는 **Base 클래스** 도입을 우선적으로 검토해야 합니다.
    *   ⚠️ **중요**: 새로운 클래스 설계 단계에서 "현재 상황에 맞는 Base 클래스를 도입하여 OOP 기반으로 설계할까요?"라고 **사용자에게 반드시 먼저 물어보고 진행**하십시오.

## 2. 작업 설계도 (Blueprints) 운영 규칙

모든 작업(기능 추가, 리팩토링 등)을 시작하기 전과 진행하는 동안에는 일관된 정보 공유를 위해 **설계도**를 작성하고 유지해야 합니다.
이 설계도는 다른 사람들이 현재까지 진행된 내용과 앞으로 할 내용을 쉽게 파악할 수 있도록 돕습니다.

*   **설계도 저장 위치**: `Docs/Blueprints/` (문서가 없다면 새로 생성)
*   **작업 절차**:
    1.  새로운 작업을 시작할 때 `Docs/Blueprints/` 폴더 아래에 새 작업용 마크다운 파일(예: `작업명_blueprint.md`)을 생성합니다. (기본 템플릿: `Docs/Blueprints/template.md` 참고)
    2.  설계도에 현재 상태, 작업 목표, 세부 구현 계획, 향후 일정을 기록합니다.
    3.  작업을 진행하면서 변경되는 사항은 즉시 해당 설계도 파일에 업데이트합니다.

## 3. 빌드 및 개발 환경

*   **타겟 프레임워크**: .NET Framework 4.8
*   **언어**: C# 및 XAML
*   **빌드 명령어**: `msbuild SJMagic\SJMagic.csproj` (또는 Visual Studio / 호환 IDE 사용)
*   **코딩 스타일**: C# 기본 네이밍 컨벤션(Class/Method는 PascalCase, 필드/변수는 camelCase 등)을 준수합니다.
